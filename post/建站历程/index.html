建站历程

为什么要建立这个网站

很长一段时间里,我使用计算机都是为了有且仅有的一个目的:打游戏.

但是,在大约初中开始,我就开始有了用计算机做一些更加有意义的事(?)的愿望.

起初,我看到网络上有许多人发布了自己的视频,于是我就想搞点视频,开始接触一些剪辑软件.但我很快就放弃了,我发现无论是硬件要求还是自己的能力都无法满足这个目标.直到现在,在这一方面还局限于做一些小的项目,比如班级上给同学老师剪点歌或者做做课件辅助.

我第二个开始尝试的项目是音乐.FL 大约是我第一个接触的软件(由于思杰马克丁的存在,这个软件基本上没法用),然后是 FreePiano.我发现自己也难以在这个方面迅速做出什么成果,而且我也没有任何的乐理知识,于是暂时放下了.

到现在,受教科书启发,我又开始搞网站了.这一次我大概算是找到能有点成果的方向了.

建立这个网站的过程

开始–建站工具的选择

建立一个网站一共需要两类工具:

  1. 服务器
  2. 网页编辑器

首先是服务器的选择.在国内大环境下, ipv4 的地址显然是不够用的,但 ipv6 还没普及,于是国内的电信,联通等公司默认都不给用户提供公网 IP ,而要用户申请才提供,而且自建服务器需要一对昂贵的设备.对于我一个小小的建站者这显然太奢侈了.然后是各厂家提供的云服务.网络上有可以花钱买的服务器,但我显然没那个钱,而且出钱买一个常年不会有人访问的服务器也太奢侈了.最后,我来到了 Github Pages .这个服务器是免费的,但只能用静态网站,不能用现在最常用的动态网站,不能搞点表单之类的东西,但够用了.如果有需要,也可以很方便地迁移到国内的 Gitee 上.

讲完了服务器,我们来讲网页编辑软件.

编辑网页的软件有三类:

  1. 纯静态网页编辑器
  2. 动态网页编辑器
  3. 静态网页生成器

在高中信息技术选修三的建站教程当中,作者给我们提供的工具是微软的 FrontPage .但工具自从发布第一个大版本后就再也没有推出过新版本,永远停留在了 2003 年.在现在这个发展迅速的年代,用 FrontPage 做出来的网页显得简陋而落后.于是我放弃了 FrontPage .而且 FrontPage 是纯静态的,在有大量信息产生的现在,我们要求我们的网页要能够批量生成,就这个角度看, FrontPage 也不能满足要求.

网络上主流的网页生成器都是针对动态网站的,如 WordPress ,是 Github Pages 不支持的,所以也放弃了.

于是我们就只有唯一一个选择:静态网页生成器.在寻找静态网页生成器的道路上,我看见过很多的名字,比如 Jekyll , Hexo , VuePress 等.但它们都需要在计算机上设置安装大量对我(一个零编程基础的菜鸟)而言太复杂软件环境,也被放弃了.

最后我来到了 Hugo .这个软件的本体只有一个 exe 文件,没有前面提到过的软件那么复杂,而且功能较完备,对我而言已经是能够找到的最好的软件了.

发展–建站的思路

最开始,设想中的建站是用 Typora 一个网页一个网页地做,但很快我就发现这样是不现实的:如果只有寥寥几个网页还好,但如果需要大量的网页的话很是麻烦,而且很容易出错,也不好维护和迁移.

这个时候我就开始寻找静态网页生成器.如上文一样,我最后选择了 Hugo .

但这个软件也不是很好使用,它需要用 cmd.exe 运行才能工作,而没有任何的图形界面,这对我又是一个困难.(后来我明白了,这个软件要搭配要 VSCode 这样比较专业的编程软件才好使.在这样的环境下,如果有图形界面反而更是麻烦.)

然后,把 hugo.exe 所在目录添加到计算机的环境变量,这样可以让 cmd 找到 hugo 的位置.然后生成一个工程目录,添加主题(theme),新建文章

hugo new post/test.md

编辑一下 config.toml 文件.

最后执行

hugo

指令.我们的需要的网页就可以生成到 public 目录中了.

结束-新的开始–建立一个网站

在 Github 当中先建立一个网页项目,然后 clone 到本地,将刚才生成到 public 下的文件全部复制到该项目下,最后上传,一个位于互联网角落的角落的小网站就生成诞生了!

感言

从最开始的纯静态站点到最后这个有模有样的简单网页,共经过了五天的时间.

在这五天的时间里,我在国内外相关网站里面来来回回,尤其是 Github 使用得特别多.虽然 Github 已经有十几年的历史了,而且中国人在里面也特别多,但直到现在(2202.1.26)为止,这个网站也没有提供任何的中文翻译,以后大概也不会有,这让我贫瘠的英语水平发挥了一点应该的作用.在此特别感谢我的英语老师!
